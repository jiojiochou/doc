    变量的声明

    # 变量的声明

    完整的声明格式

    声明了类型后TypeScript就会进行类型检测, 声明的类型可以称之为类型注解(Type Annotation)

    var/let/const 标识符: 数据类型 = 赋值;

    # 类型推导

    在ts中我们可以省略类型注解, 它会根据值的类型推断出类型

    let进行类型推导, 推导出来通用类型, const进行类型推导, 推导出来字面量类型

    let name = 'chenchu'; 👈 // let name: string
name = 666; 👈 ❌❌❌ // 不能将类型“number”分配给类型“string”。
let age = 18; 👈 // let age: number
const height = 1.88; 👈 // const height: 1.88

    # unknown类型

    unknown是TypeScript中比较特殊的一种类型, 它用于描述类型不确定的变量

    和any类型有点类似, 但是unknown类型的值上做任何事情都是不合法的

    let id: unknown = 123;
id = 'chenchu'; 👈 ✔✔✔ // 可以这样操作
console.log(id.length); 👈 ❌❌❌ // 不可以进行任何操作

    # 类型缩小

    let id: unknown = 123;
id = 'chenchu'; 👈 ✔✔✔ // 可以这样操作
if(typeof id === 'string'){
    console.log(id.length); 👈 ✔✔✔
}

    # void类型

    void通常用来指定一个函数是没有返回值的, 那么它的返回值就是void类型

    如果返回值是void类型, 那么我们也可以返回undefined类型(js中函数没有明确返回值, 那么默认返回undefined)

    function foo(num1: number, num2: number): void{
    console.log(num1+num2);
    return undefined
}
foo(100, 100)

    # never类型

    never表示永远不会发生值的类型, 比如一个函数

    如果一个函数中是一个死循环或者抛出一个异常, 那么这个函数会返回东西吗?👇👇👇

    不会, 那么写void类型或者其他类型作为返回值类型都不合适, 我们就可以使用never类型

    # 泛型

    泛型就是给我们的代码增加一种相对宽泛的类型约束

    将类型像函数参数一样传递

    function foo<T>(arg: T): T{
  return arg
}
const res1 = foo<string>('zzz')
const res2 = foo<number>(666)
const res3 = foo<{name: string, age: number}>({name: "ccc", age: 18})

let res4 = foo('aaa')
let res5 = foo(9999)
let res6 = foo({name: 'fff', age: 66})
let res7 = foo<any[]>([]) // 有时候类型推断错误, 最好还是明确指定 (`never[] 👉 any[]`)

    # 传入多个类型

    function foo<T, E>(arg1: T, arg2: E){
    // ...
}
foo(666, 999)

    # 泛型类

    class Point<T = number>{
	x:T
    y:T
    constructor(x: T, y: T){
        this.x = x
        this.y = y
    }
}
const p1 = new Point(100, 100)
console.log(p1.x)
const p2 = new Point<string>('666', '999')
console.log(p2.x, p2.y)

    # 泛型约束

    interface Ilength {
  length: number
}
function foo<T extends Ilength>(arg: T) {
  return arg
}
const res1 = foo<string>('xxxx')
const res2 = foo(['abc', 'cba', 'aaa'])
const res3 = foo({length: 18})
const res4 = foo(123) 👈 ❌❌❌ // 类型“number”的参数不能赋给类型“Ilength”的参数。

    # 泛型参数约束

    function foo<O, K extends keyof O>(obj: O, key: K){
    return obj[key]
}
const info = {
    name: 'csx',
    age: 18,
    height: 1.88
}
const name = foo(info, 'name')

    # 映射类型

    有的时候，一个类型需要基于另外一个类型，但是你又不想拷贝一份，这个时候可以考虑使用映射类型

    映射类型建立在索引签名的语法上

    interface IPerson {
    name: string
    age: number
}
type MapType<Type> = {
    [property in keyof Type]: Type[property]
    // name: string 依次遍历一遍
    // age：number
}
type NewPerson = MapType<IPerson>

    # 映射类型修饰符

    type MapPerson<Type> = {
    // 删除 readonly, 删除可选`?`
    -readonly [Property in keyof Type]-?: Type[Property]
}

    # 条件类型

    条件类型的写法有点类似于 JavaScript 中的条件表达式 (condition ? trueExpression :falseExpression)

    type IDType = number | string
type ResType = number extends IDType ? true : false //type ResType = true

    # 例子

    function sum(num1: number, num2: number): number
function sum(num1: string, num2: string): string
function sum(num1, num2){
    return num1 + num2
}
// 使用条件类型 👇👇👇
function sum<T extends number | string>(num1: T, num2: T): T extends number ? number : string
function sum(num1, num2){
    return num1 + num2
}

    # 分发条件类型

    如果我们在 ToArray 传入一个联合类型，这个条件类型会被应用到联合类型的每个成员:

    当传入string|number时，会遍历联合类型中的每一个成员;

    相当于ToArray | ToArray;

    所以最后的结果是: string[] | number[];

    type toArr<T> = T extends any ? T[]:never
// 先用 number 执行一次
// 再用 string 执行一次
type newType = toArr<number | string> // number[] | string[]

baz(100);

baz.age = 100;

    # 函数重载

    减少因参数类型不一样而重新声明函数

    function add(num1: number, num2: number): number
function add(num1: string, num2: string): string
function add(num1: any, num2: any): any { 👈 // 通用函数不能被调用
  return num1 + num2
}
add(100, 100)
add('zzz', 'xxx')
add({asd: 'sds'}, 'hhf') 👈 // 没有与此调用匹配的重载

    # 函数中this

    函数中的this是具有隐式类型any的, 需要明确类型

    作为函数的第一个参数, 用于声明this的类型(名词必须叫this)

    在后续调用函数传入参数时, 从第二个参数开始传递的, this参数会在编译后被抹除

    function foo(this: {name: string}){ 👈 // 作为函数的第一个参数, 用于声明this的类型(名词必须叫`this`)
	console.log(this)
}

    # TS中类型检测-鸭子🦆类型

    鸭子🦆类型:如果一只鸟, 走起来像鸭子, 游起来像鸭子, 看起来像鸭子, 那么你可以认为它就是一只鸭子

    鸭子🦆类型:只关心属性和行为, 不关心你具体是不是对应的类型

    class Person{
    constructor(public name: string, public age: number) {}
}
class Animal{
    constructor(public name: string, public age: number) {}
}
class Dog{
    constructor(public name: string, public age: number) {}
}
function foo(fn: Person){
	return new fn('zs', 18)    
}
foo(Person) 👈 ✔✔✔
foo(Animal) 👈 ✔✔✔
foo(Dog) 👈 ✔✔✔
