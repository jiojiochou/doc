    # html+css

    # !DOCTYPE html 标签

    ​ 1.告诉浏览器是 HTML5页面

    ​ 2.告诉浏览器用 HTML5的标准去解析识别内容

    ​ 3.必须放在HTML文档的最前面, 不能省略, 省略会出现兼容问题

    ​ 4.单标签不必再写 '/' 了

    # html标签

    ​ 1.根标签, 顶级元素

    ​ 2.:root(伪类)可以代表 html 标签

    属性:

    # meta标签

    ​ 1. 可以用于设置网页的字符编码

    # img标签

    1. src属性: source(来源)单词的缩写, 图片的来源, 可以是网络地址, 可以是绝对路径, 可以是相对路径
2. alt属性: 加载不出来图片的时进行文本显示, 有利于屏幕阅读器描述这个图片

    # a标签

    ​ 1. href 属性: #id, .html(会跳转到这个页面), .zip(会下载), http://www.baidu.com(会向服务器请求这个index.html 并缓存到浏览器). 可以是 mailto: 83553175@qq.com, mailto(协议) 会调用邮箱(应用程序)

    ​ 2. taget属性: 页面的打开方式(本页面打开, 新建标签页打开); value: _self(本页面), _blank(新建页), _top(结合iframe使用), _prent(结合iframe使用)

    # video标签

    • 早期网页只展示图文, 没有相应的解决方案 基本使用flash代替, 但各个浏览器的兼容版本不同不是很友好

    • 用于HTML或者XHTML文档中嵌入媒体播放器, 用于支持文档内的视频播放器

    • video常见的属性

      常见属性 值的方式 属性作用
      src URL地址 视频播放的URL地址
      width pixels(像素) 设置video宽度
      height pixles(像素) 设置video高度
      controls Boolean类型 是否显示控制栏, 包扩音量, 跨帧, 暂停/恢复播放
      autoplay Boolean类型 是否视频自动播放 (某些浏览器需要添加muted, 比如chrome)
      muted Boolean类型 是否静音播放
      preload none/metadata/auto 是否需要预加载视频, metadata表示预加载与数据(比如视频时长等)
      poster URL地址 一海报帧的URL
      <!-- 基本使用 -->
<video src="./xxxx.MP4" width="600" controls muted></video>
<!-- 兼容写法 -->
<video src="./xxxx.MP4">       <!-- 依次进行匹配 -->
    <source src="./xxxx.ogg">
    <source src="./xxxx.webm">
    <source src="./xxxx.xxx">
    <p> 当前您的浏览器不支持视频的播放, 请更换更好的浏览器! </p>
</video>

    # css常见选择器(selector)

    1. 属性选择器

      [title] {
	// 属性选择器, 有属性 title 的会被选择上
}
[title=bobodi]{
	// 有title属性的 且 title属性的值是 'bobodi' 的元素会被选中
}
// 了解-----👇
[attr*=val]{
	// 属性值包含某一个值
}
[attr^=val]{
	// 属性值以val开头
}
[attr$=val]{
    // 属性值以val结尾
}
[attr|=val]{
	// 属性值等于val或者以val开头后面紧跟连接符-;
}
[attr~=val]{
    // 属性值包含val, 如果有其他值必须以空格和val分割; (val aaa || val bbb)
}

}

    # 标准流(Normal flow)

    • 默认情况下,元素都是按照normal flow
    • 从左到右, 从上到下按顺序摆放好
    • 默认情况下, 互相之间不存在层叠现象

    # position(定位)

    • 常用取值: static | relative(相对定位) | absolute(绝对定位) | sticky(粘性定位) | fixed(固定定位)
    • 元素默认值: static (非定位元素)
    绝对居中
/*{	
    position: relative;
    transform: translate(-50%);
    margin-left: 50%;
}*/
relative 定位参照对象是元素自己原来的位置, 还占据原来的空间
fixed 定位参照对象是视口(viewport)
absolute 定位参照对象是最临近的定位祖先元素, 如果没有找到这样的祖先元素, 参照对象是视口
设置成 fixed or absolute的行内块(inline block box)元素, 可以随意设置宽高, 宽高默认是内容撑起的
    • sticky
      • 可以看做是相对定位和固定(绝对)定位的结合体;
      • 他允许被定位的元素表现得像相对定位一样, 直到它滚动到某个阈值点;
      • 但达到这个阈值点时, 就会变成固定(绝对)定位;
      • sticky是相对于最近得滚动祖先包含视口的

    # z-index

    • 取值可以是正整数, 负整数, 0
    • 如果是兄弟关系
      • z-index越大, 层叠在越上面
      • z-index相等, 写在后面的那个元素层叠在上面
    • 如果不是兄弟关系
      • 各自从元素自己以及祖先元素中, 找出最邻近的2个定位元素进行比较
      • 而且这2个定位元素必须有设置z-index的具体数值

    # float(浮动)

    • 其属性可指定一个元素沿其容器的左侧或者右侧放置, 允许文本和内联元素环绕它
    • 取值: none(不浮动, 默认值) | left(向左浮动) | right(向右浮动)
    • 会脱离标准流!!!!
    • position > float > 标准流 (之间的层叠关系)
    • 浮动元素之间不会层叠
    • 水平方向剩余的空间不够显示浮动元素, 浮动元素将向下移动, 直到有充足的空间为止
    • 浮动元素不能与行内级内容层叠, 行内级内容将会被浮动元素推出

    # flex (弹性布局)

    两个重要概念:

    • 开启了flex布局的元素叫flex container;
    • flex container里面的直接子元素叫做 flex item;

    当flex container中的子元素变成了flex item时, 具备一下特点:

    • flex item的布局将受flex container属性的设置来进行控制和布局;
    • flex item不再严格区分块级元素和行内元素;
    • flex item默认情况下是包裹内容的, 但是可以设置宽度和高度

    设置display属性为 flex 或者inline-flex可以成为 flex container

    • flex: flex container 以block-level形式存在
    • inline-flex: flex container 以 inline-level 形式存在

    flex相关的属性

    flex相关属性 描述
    flex-flow 简写属性
    flex-direction 决定主轴的方向
    flex-wrap 默认是一行显示, 决定元素是否换行
    justify-content 主轴上的分布方式
    align-items 交叉轴上的分布方式
    align-content

    flex-item的相关属性

    flex-item的相关属性 描述
    order 可以设置任意整数(正整数, 负整数, 0), 值越小就越排在前面, 默认是0
    align-self 单独设置某一个元素的排布位置
    flex-grow (拉伸/成长) 默认值0, 可以设置任意非负数字 (正小数, 正整数, 0)
    flex-shrink (缩小) 默认值1, 可以设置任意非负数字 (正小数, 正整数, 0)
    flex-basis 不会压缩元素, 会让内容显示完整
    flex缩写属性 flex-grow || flex-shrink || flex-basis 的简写, flex属性可以指定1个, 2个或3个值

    决定flex-item 最终base size的因素, 从优先级高低

    • max-width\max-height\min-width\min-height
    • flex-basis
    • width\height
    • 内容本身的size

    # media媒体查询

    • 可以根据设备的类型 (比如屏幕设备, 打印机设备) 或者特定的特性 (比如屏幕的宽度) 来修改你的页面

    • 媒体查询的使用方式主要有三种:

       /* import 结合 媒体查询 */
 /* @import url(./index.css) 媒体查询 */

      <link rel="stylesheet" media="(max-width: 500px)" href="./index.css"/> or
<!-- 每条媒体特性表达式都必须用括号括起来 -->
<link rel="stylesheet" media="screen and (max-width: 500px)" href="./index.css"/>
      • 方式三: 通过 @media 使用不同的css ( 常用 ) ;
      @media (max-width: 500px) {  /* 条件达成 执行{}里面的代码块 */
    body {

      screen 主要用于屏幕
      speech 主要用于语音合成器

    • 布局视口
      • 默认情况下, 一个在pc端的网页在移动端会如何显示
        • 第一, 它会按照宽度为980px来布局一个页面的盒子和内容;
        • 第二, 为了显示可以完整的显示在页面中, 对整个页面进行缩小;
      • 我们相对于980px布局的这个视口, 称之为布局视口 (layoutviewport)
        • 布局视口的默认宽度是980px;

    移动端适配方案

    • 方案一: 百分比设置;
      • 因为不同属性的百分比值, 相对于的可能是不同的参照物, 所以百分比往往很难统一;
      • 所以百分比在移动端适配中使用是非常少的
    • 方案二: rem单位+动态html的font-size;
    • 方案三: vw单位;
    • 方案四: flex的弹性布局;
    • 方案五: 使用淘宝rem适配库 (opens new window); viewport单位已经得到众多浏览器的广泛使用, 此方案可以弃用
